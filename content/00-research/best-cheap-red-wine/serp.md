## 3. Content Gap Analysis

Based on all 7 competitors read:

### Gaps we can own outright

1. **No competitor organises cheap red wines by style/structure (light vs. medium vs. bold).** Every competitor is either flat-list or variety-by-region. Nobody has created "Best Cheap Light Red", "Best Cheap Medium-Bodied Red", "Best Cheap Bold Red" as H2 buckets. This is the most natural way to help a reader choose.

2. **No competitor addresses the "cheap red for different occasions" angle in a structured way.** TheWineMarket comes closest with its "Great For:" tags but doesn't elevate these into H2 sections. A dedicated section or matrix of "cheap red by occasion" (weeknight dinner, dinner party, BBQ, cooking, gift) is unclaimed territory.

3. **No competitor covers value regions systematically.** WineFolly gestures at it but is stale. Nobody has a "Why Cheap Red Wine from These Regions Overdelivers" section covering Côtes du Rhône, Languedoc, Douro, Mendoza, Ribera del Duero, McLaren Vale — the regions that reliably punch above their price. This is practical buyer intelligence no current ranking page provides.

4. **No competitor has serving temperature guidance for cheap red wine.** Several cheap reds are better served slightly chilled (10–12°C) — Beaujolais, lighter Côtes du Rhône — but no page mentions this. This is an E-E-A-T differentiator.

5. **No competitor has a "What makes cheap red wine taste cheap?" explainer** — the structural faults (excessive sulfites, residual sugar, thin acidity, rough tannins from young vines) that distinguish bad-cheap from good-cheap. Real Simple gestures at it via sommelier quotes but doesn't synthesise it into actionable guidance.

6. **No competitor has a body/tannin description per wine.** TheWineMarket's "Great For:" tags are use-case oriented. Food & Wine has food pairings. Nobody maps each wine on a light-to-bold spectrum with explicit structure notes.

7. **WineQuiz advantage unclaimed by every competitor.** Not one competing page has an interactive recommendation tool. Our existing WineQuiz component already does this.

### Gaps the SERP signals via PAA and related searches

- "What wine is best for GERD?" — acid-sensitive readers. No competitor addresses acid-management in cheap reds.
- "What is the best value red wine?" — value framing. WineFolly comes closest but stale.
- "Which red wine is good and affordable?" — beginner-entry angle. Some competitors address this but no one with depth.

## 6. Novel Feature Recommendation

### Primary recommendation: "Cheap Red by Body + Occasion" two-axis chip matrix

**What it is:** An inline reference table (no new component needed — can use existing prose table or a lightweight `WineTip` callout box) that maps each of our 12 wines on two axes:
- **Body** (Light / Medium / Bold)
- **Best Occasion** (Weeknight / Dinner Party / BBQ / Cooking / Gift)

This creates a quick-scan visual that no competitor has. A reader can look at the matrix and immediately pick "I want a bold red for BBQ under $20" without scrolling through 12 individual wine entries.

**Implementation:** A simple 12-row Markdown table placed immediately before the wine listings (inside a `WineTip` callout or as a standalone reference table):

| Wine | Body | Best For | Price |
|------|------|----------|-------|
| [Wine 1] | Light | Chilling, Picnics | $XX |
| [Wine 2] | Medium | Weeknight Dinner | $XX |
| ... | ... | ... | ... |

This table is:
- Zero additional component cost (uses existing Markdown table rendering)
- Absorbs NeuronWriter entities (body types, occasion uses)
- Genuinely useful — first thing in the SERP to answer "which cheap red for X"
- Directly builds on our WineQuiz advantage (quiz = interactive version of same logic)

### Secondary recommendation: Per-wine "serve at" temperature tag

A small inline note per wine description: "Serve at 14–16°C" for medium/bold reds, "Serve at 12–14°C" for lighter styles. No competitor has this. Adds practical value, signals winewriting authority, and is zero component cost (inline prose).

### Tertiary (for Dae queue): "Value radar" scatter plot

Critic score (Y-axis) vs. price (X-axis) for all 12 wines. Visualises which bottles over-deliver per dollar. Lines up with the master audit's `<DotScaleRow>` primitive already in Dae's backlog. Tag for Dae at brief stage.
