## Gap Analysis

### What every listicle competitor has (and we need to match)
1. **Per-wine H2/H3 headings** — Delish (8), Fine Dining Lovers (9), Tasting Table (10), LoveToKnow (12) all structure individual wines as headings. We must do this.
2. **Multi-style coverage** — The top listicles cover red, white, and at least some rosé/sparkling options. Red-only is a gap (FDL's weakness). We should cover red + white + rosé + sparkling as H2 buckets.
3. **Specific bottle names** — Tasting Table and Delish recommend actual bottles. LoveToKnow and FDL only go to varietal level. Specific bottles = more purchase intent captured.

### What nobody currently has (our gap lane)
1. **Critic/professional scores per bottle** — Zero competitors show scores. We carry this natively via `<AffiliateCTA>` with JS, WE, WS ring charts.
2. **Per-wine tannin + acidity suitability rating for sangria specifically** — No one explains WHY each wine works at the structural level (tannin level 1-5, acidity level 1-5, how it interacts with the sugar/fruit/spirit mix).
3. **Occasion + season matrix** — Wine Insiders has seasonal recipes but no wine-specific mapping. Nobody maps specific bottles to occasions (Summer BBQ / Holiday Party / White Wine Brunch / Date Night Rosé).
4. **Interactive WineQuiz** — We have this. Zero competitors do.

## Novel Feature Recommendation

**Shippable novel feature: Per-wine Sangria Suitability Strip**

No competitor currently explains — at the individual wine level — why a wine works structurally in sangria. They just say "Tempranillo is good" or "avoid oaky Chardonnay."

We should add a 4-column inline data strip under each wine's H3 heading:

| Column | Data |
|--------|------|
| Tannin Level | Low / Medium / High (1-5 dot scale) |
| Acidity | Bright / Medium / Soft (1-5 dot scale) |
| Sweetness | Dry / Off-dry / Semi-sweet |
| Sangria Style Best For | Red Classic / White Citrus / Rosé Fruit / Sparkling |

**Format:** Pure Markdown table under each H3 — no new component needed, no Dae involvement required at Stage 1. Optionally upgrade to `<WineDataBlock>` primitive later (same Dae backlog).

**Why it wins:** It directly answers the buyer's implicit question ("but WHY does this wine work for sangria?") in scannable form — something no editorial competitor has. It differentiates at the individual wine level and absorbs NeuronWriter entity terms naturally (tannins, acidity, residual sugar, sangria characteristics).

**Specific columns:** Wine name / Tannin Level (1-5 dots) / Acidity (1-5 dots) / Best Sangria Color (Red / White / Rosé / Sparkling) / Sangria Occasion (BBQ / Brunch / Holiday / Date Night)
