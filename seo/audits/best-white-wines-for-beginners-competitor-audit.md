## Median competitor wine count

Ranging 5, 5, 5, 8, 9, 11, 14, ~40 → **median ≈ 8**.

We ship **8 wines**. **We are at the median.** Competitors with higher counts (Wine Folly at ~40, YourWineStore at 14) win on breadth-as-reference, not on focused picks; competitors with lower counts (Wine Ninjas, Basta Pasta, Greatist) win on clarity/infographic visuals. Our 8 specific SKU picks sits in the right zone for "buying guide" intent, which is what this keyword rewards at the commercial tier.

## Gap analysis — what top pages have that we don't

1. **Visual style-category infographic.** Wine Folly's 5-category infographic and Greatist's 4-category infographic are the standout visual assets on the SERP. Both get shared and re-pinned heavily. We have zero visual summary of which of our 8 wines maps to which style (crisp / aromatic / full / sweet).
2. **"How to identify this style" quick-ID blocks.** Wine Folly's category blocks open with a plain-English "how to identify this style" block (body weight, acidity, typical flavours, ideal food). Greatist uses the same pattern. We introduce each wine one-at-a-time but never step back to show the reader the 4 style lanes side-by-side in one glance.
3. **Key takeaways summary at top of article.** Greatist opens with a 3-bullet "Key takeaways" box above the fold. Useful for readers who skim. We go straight into prose.
4. **A "start here" pick explicitly tagged for the absolute beginner.** Wine Ninjas names Pinot Gris as the #1 beginner pick and defends it. Lexi's list puts Vinho Verde first. We have a TopPicksInline with 3 picks but no explicit "if you've never had white wine before, open this one first" signpost.
5. **Sweetness scale / residual sugar indicator per wine.** None of our competitors have this either. Noted under novel opportunity below.
6. **Lighter/heavier body indicator per wine.** Same — nobody on the SERP does this with a visual scale.

## Novel opportunity (at least one feature no competitor has)

**Add a sortable 8-row "White Wine Beginner Style Matrix" table above the wine listings**, with these exact columns:

| Column | Value range | Source |
|---|---|---|
| Wine | link to our H2 | — |
| Grape | Pinot Grigio / Sauv Blanc / Riesling / Chardonnay / Moscato | frontmatter |
| Sweetness | Bone-dry / Dry / Off-dry / Sweet | editorial judgment, noted in each wine's description |
| Body | Light / Medium / Full | editorial judgment |
| Best food | 1 short phrase (salads / spicy / roast chicken / dessert) | existing pairings field |
| Price | ~$ band ($15–20 / $20–25 / $25–30) | existing research notes |

One row per wine, 8 rows total. Render as a static HTML table (not a new Astro component — static is fine for v1, keep scope small). Anchor link each wine name to its H2 below. This gives the reader at-a-glance mapping from "I want something dry and light with seafood" → "Santa Margherita Pinot Grigio" without scrolling through 8 H2s.

No competitor on the SERP combines sweetness + body + food + price in one scannable table against named SKUs. Wine Folly's infographic maps varietals to style categories but stops there; Greatist's infographic is prose-style, not sortable; everyone else just lists wines.

**Second novel opportunity (secondary, if scope allows):** add a lightweight "sweetness meter" inside each AffiliateCTA — a small 5-dot scale under the star rating where e.g. Santa Margherita shows 1/5 filled (bone-dry), Eroica Riesling shows 3/5 (off-dry), Damilano Moscato shows 4/5 (sweet). Would require an AffiliateCTA prop addition, so more work than the table. Flag for Lia + component author if she thinks it's worth it.
